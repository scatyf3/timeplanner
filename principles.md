# 原则库 —— TimePlanner 的「价值观」

> v1 直接从日记模板 + 收工记分板抽取（PeriodicNotes/Templates/Daily.md）。
> agent 排 plan、做 reflect 时以此为准。之后逐步和 agent 对话补充、二期迁 Cubox 高亮。

## 四个工作块（每块「到点物理硬停」）

- **main** —— 科研 / 带 deadline 的正事
- **side** —— 写代码 / PhD 申请调研
- **生活** —— 吃饭、杂务、社交、回血
- **健身 / 娱乐** —— 身体那一格

planner 每天的工作 = 把这四块摆进日历，各自封顶、到点硬停。

> 写日历时每段标一个 bucket，用于配色（对齐 Time record）：
> `main`/`side`＝工作·蓝、`life`＝生活杂务·黄、`health`＝健康·浅绿、`fun`＝娱乐·深绿。
> 「健身/娱乐」这一格按当下活动选 `health` 或 `fun`。

## 核心原则

1. **每日独立预算，互不相吞，各自封顶。** 一格的时间不能被另一格吞掉。赢 =「可控产出存在」，不是「做完了没」。
2. **到点物理硬停，不看做没做完。** 专注 block 默认 90min，到点就停。停手前**先写下 next action 再合电脑**——脑子才肯卸载后台。
3. **专注 block 是骨架。** 模板默认一天 3 个 90min block（可按 workload 调 2–4）。晨间 triage ≤15min，产出是一张排好序的单子，不是反复 processing。
4. **盯 agent 有上限。** 单次 ≤15min，全天合计 ≤30min；之后靠通知兜底，不持续盯。
5. **生活 / 身体 ≥1 个小单元。** 赢 = 它发生了，别和过去峰值比。
6. **纯复盘 ≤20–30min，且必须落一行 takeaway 才算数。** 只打转、不产出 → 走 cap，停。
7. **meta-work ≠ 复盘。** 改 workflow / 系统 / 模板这种**有产出**的事算工作，占一个专注 block、走专注预算，结束**必须 ship 一个具体改动**。连两次都在改 workflow 不做正事 = 逃避换马甲，停。
8. **天气自适应。** 热天户外挪早晚 / 去室内地下健身房；天气舒适就把「生活/身体」那格放户外。

## 收工 4 问（reflect 记分板）

- ① 专注 block 发生了？
- ② 按时停了？（停手前写了 next action？）
- ③ 生活 / 身体那个小单元做了？
- ④ 纯复盘落成 takeaway 了？

## planner 的硬约束（辅助式 + 不添乱）

- **永不覆盖外部事件。** 没带 `timeplanner` 标的日历事件一律当固定约束，只在其空隙里排自己的块。
- **写日历前必过确认。** 先给 diff，你点头才落地（dry-run gate）。
- **单一真相不混写。** GCal = 意图（Plan/Actual），ActivityWatch = 现实，二者永不混写。
- **workload 推断先简单、可解释、事后校准，别过早上模型。**
