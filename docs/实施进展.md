# TimePlanner 实施进展

> 配套 [[time planner - 实施计划]]。本文记录已落地的实现 + 变更日志。
> 截至 2026-07-08：**Phase 0 的 M0–M3（本地版）全部跑通**，尚未接 Google / Telegram。

## 一句话现状

一个能跑的独立 Python 应用：`timeplanner` CLI 读你的笔记 / ActivityWatch / 天气，
agent 出一天的 plan 草案，**你确认才写入本地 timeline**（schema 与 GCal 一致），
晚间可 reflect，并把规划思考/原则攒进 planner 记忆。GCal 与 TG 是后续「换壳」。

## 已实现的结构

```
timeplanner/
├─ timeplanner/
│  ├─ config.py          # .env 配置（库路径/AW/天气/GCal/TG + cache/data 目录）
│  ├─ cli.py             # doctor / summary / plan / confirm / log / reflect / memory
│  ├─ agent.py           # Agent SDK loop + 系统 prompt + 7 个工具 + 终端渲染
│  └─ core/
│     ├─ notes.py        # 解析日记/project → TODO/deadline/workload（实时读）
│     ├─ activitywatch.py# 本地 AW REST → 在机时长/专注 block/Top 应用
│     ├─ weather.py      # Open-Meteo → 户外建议
│     ├─ gcal.py         # GCal 读写骨架：timeplanner 标记 + dry-run（待 OAuth）
│     ├─ timeline.py     # 本地 Plan/Actual timeline（同 gcal.Event schema）
│     ├─ memory.py       # planner 记忆：规划思考 + 候选原则
│     └─ cache.py        # 通用 mtime 缓存原语（现供 memory 用）
├─ principles.md         # 原则库 v1（从日记模板抽取）
├─ data/                 # 本地 timeline：plan.json / actual.json（gitignore）
└─ .cache/               # planner 记忆：memory.json（gitignore）
```

## 命令速查

| 命令 | 作用 |
|---|---|
| `timeplanner doctor` | 环境自检（库/AW/天气/GCal/SDK） |
| `timeplanner summary [--date]` | 只读汇总：笔记 workload + AW + 天气 + 本地 timeline |
| `timeplanner plan [--date]` | agent 出草案并 stage（需 Agent SDK） |
| `timeplanner confirm [--date] [--yes]` | 辅助式闸门：预览 diff / `--yes` 写进本地 Plan |
| `timeplanner log START END BUCKET SUMMARY…` | 录一条 Actual 事件（②自报层） |
| `timeplanner reflect [--date]` | 晚间复盘：①Plan vs ②Actual vs ③Observed |
| `timeplanner memory [--clear]` | 看/清 planner 记忆 |

每日循环：早上 `plan → confirm --yes`；白天做完一件 `log`；睡前 `reflect`。

## 关键设计沉淀

- **换壳哲学落到存储层**：`timeline.py` 与 `gcal.py` 共用同一套 `Event` + 同格式 summary，
  以后配好 OAuth，换存储后端即可，core/agent 一行不动。
- **辅助式闸门**：agent 永不擅自写；`plan`（stage）→ `confirm`（看 diff）→ `--yes`（落地）。
- **两级原则**：`principles.md`（git 追踪的源头真相）＋ `.cache/memory.json`
  （agent 积累的候选，你觉得好的手动并入）。
- **只读数据源**：Obsidian 库全程实时读、零副本、改即生效。
